## Architecture Overview

Single-page React application that reads live task data from the Cowork `scheduled-tasks` MCP at load time and on manual refresh. All state is in-memory (no backend DB needed). Rendered as a `.jsx` artifact inside the Cowork session — same pattern as CommandCenter.jsx.

## Tech Stack

| Layer | Technology | Rationale |
|-------|-----------|-----------|
| Frontend | React (JSX) | Already used for Command Center; renders directly in Cowork |
| Data | scheduled-tasks MCP (`list_scheduled_tasks`) | Live source of truth for all task data |
| Cron Parser | Custom JS function | Lightweight — no npm needed for simple patterns |
| Styling | Inline styles (dark theme) | Consistent with Command Center aesthetic |
| Hosting | Cowork artifact renderer | No deployment needed for MVP |

## Key Components

1. **`CronVisualizer` (root)** — Fetches task list on mount, holds all state, renders the full UI
2. **`TaskCard`** — Displays one scheduled task: name, schedule, status badge, next/last run times, enable toggle, trigger button
3. **`CronParser`** — Pure JS function that converts cron expressions to human-readable strings (e.g. `*/30 * * * *` → "Every 30 minutes")
4. **`StatusBadge`** — Color-coded pill: ACTIVE (green), IDLE (yellow), ERROR (red), NEVER RUN (gray)
5. **`HeaderBar`** — Title, live task count, last-refreshed timestamp, manual refresh button

## Data Model

Data sourced from `list_scheduled_tasks` MCP. Each task object:

| Field | Type | Notes |
|-------|------|-------|
| `taskId` | string | Unique identifier |
| `description` | string | Human-readable task name |
| `schedule` | string | Human-readable schedule from MCP |
| `cronExpression` | string | Raw cron string |
| `enabled` | boolean | Active or paused |
| `nextRunAt` | ISO timestamp | When it fires next |
| `lastRunAt` | ISO timestamp | When it last fired |
| `fireAt` | ISO timestamp | For one-time tasks |

## CronParser — Expression Coverage

Handle the most common patterns used in Rich's task set:

| Pattern | Output |
|---------|--------|
| `*/N * * * *` | Every N minutes |
| `0 H * * *` | Daily at H:00 AM/PM |
| `0 H * * D` | Every [day] at H:00 |
| `@daily` | Every day at midnight |
| Fallback | Display raw expression |

## Technical Risks

| Risk | Mitigation |
|------|-----------|
| MCP not accessible from JSX artifact | Fetch data server-side before rendering; pass as props |
| Cron parser misses edge cases | Show raw expression as fallback — never break the UI |
| Toggle/trigger MCP calls fail silently | Optimistic UI update + error state badge on failure |
