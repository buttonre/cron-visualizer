# UX Brief — Cron Visualizer

**Date:** 2026-03-30
**Status:** Pending Review
**Author:** Ted (UX & Human Element)

---

## Who Uses This

Rich — a solo power user who runs a personal automation stack and needs to quickly check the status of scheduled tasks without digging through config files or session logs.

## Core User Flow

1. Rich opens Cron Visualizer in Cowork
2. All scheduled tasks load instantly — name, schedule in plain English, last/next run, and status at a glance
3. Rich spots a task that's overdue or errored, toggles it or manually fires it from the same screen
4. Done — no terminal, no config file, no digging

## Friction Points to Avoid

1. **Cron gibberish** — showing `*/30 * * * *` instead of "Every 30 minutes" forces Rich to mentally translate. All cron expressions must be human-readable. Raw expression shown only as a tooltip or secondary line if needed.

2. **No status without a run history** — if a task has never run, the UI should say "Never run" clearly, not show a blank or null timestamp. Every task should have a readable state at all times.

3. **Confirmation anxiety on destructive actions** — the enable/disable toggle and manual trigger button need to be distinct and require no double-click or confirm dialog for MVP. Rich is a power user. Fast action > safety theater. Reserve confirms for delete operations only.

## Interface Notes

- Match Command Center dark theme exactly — same font, same color palette, same card style
- Status indicators use the same color system: green = active/OK, yellow = idle/paused, red = error, gray = never run
- Task cards should scan vertically — most important info (name + status + next run) reads in under 2 seconds per card
- Timestamps: show relative time ("2 hours ago", "in 15 minutes") not raw ISO strings
- Keep the manual trigger button labeled **▶ RUN NOW** — not "Execute" or "Fire" or "Trigger"
- Enable/disable uses a simple toggle switch — no dropdown, no modal

## Definition of "Good Enough" for MVP

Rich can open the visualizer, see all 4 current tasks with their status and next run time in plain English, and toggle or manually trigger any of them in one click — all without leaving Cowork.
