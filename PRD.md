## Overview

Cron Visualizer is a local web-based UI for visualizing, managing, and understanding scheduled tasks (cron jobs). It targets Rich's personal Cowork automation stack — making it easy to see what's scheduled, when it last ran, when it runs next, and whether it succeeded or failed. No more guessing what the Brain Trust watcher or nightly summary jobs are doing.

## Goals

- **Primary:** Give Rich a single-screen view of all scheduled tasks with status, next run time, and last run result
- **Secondary:** Allow enabling/disabling tasks and triggering manual runs without touching config files
- **Measurable:** Reduces time spent debugging "did that task run?" from minutes to seconds

## Scope — MVP

**In scope:**
- Read and display all scheduled tasks from the Cowork scheduled-tasks system
- Show task name, schedule (human-readable), enabled/disabled state, next run time, last run time
- Color-coded status indicators (active, idle, error, never run)
- Manual trigger button per task
- Enable/disable toggle per task
- Simple cron expression parser — show "Every 30 minutes" not `*/30 * * * *`
- Responsive single-page React app

**Out of scope (v1):**
- Task creation UI (use scheduled-tasks MCP directly for now)
- Run history log / full execution transcripts
- Multi-user or cloud deployment
- Mobile app

## Success Metrics

| Metric | Target |
|--------|--------|
| All scheduled tasks visible at a glance | 100% of tasks shown |
| Cron expressions human-readable | All expressions parsed correctly |

## Constraints

- Timeline: 1–2 day build for MVP (it's a UI over existing data)
- Budget: Zero — uses existing Cowork MCP tools
- Tech: React (already in use for Command Center), scheduled-tasks MCP for data
- Runs locally inside Cowork session — no external hosting needed

## Kill Conditions

- If building this duplicates something already in the Cowork UI natively
